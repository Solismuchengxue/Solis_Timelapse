# Sony 延时摄影 WebUI

面向 Sony ZV-E10 RAW/JPEG 照片序列的本地延时摄影处理工具。它会自动识别连续拍摄段，分析亮度变化和异常帧，并以一次 JPEG 编码完成去闪、暗帧修正和风光调色，最后导出 MP4。

整个流程在本机运行。选中的原始照片目录只读，处理结果写入项目自己的工作目录，不会移动、改名或覆盖原始照片。

## 启动

1. 安装 Python 3.12，并在安装时勾选 `Add Python to PATH`。
2. 双击根目录的 `run.bat`。
3. 首次启动会自动创建 `.venv` 并安装依赖，完成后浏览器会打开 `http://127.0.0.1:9501/`。

以后只需再次双击 `run.bat`。关闭启动窗口即可停止 WebUI。

## 使用方法

### 1. 选择素材

在“工作台”中点击“选择文件夹”，选择一段或多段延时摄影照片所在的目录，然后点击“扫描素材”。

- 支持 JPG/JPEG 和 Sony ARW。
- 可以直接选择项目外部的照片目录，无需复制到本项目。
- 扫描会按照片拍摄时间和时间间隔自动分段。

### 2. 检查分段

左侧会列出识别出的延时段。可以调整段名、顺序和边界，也可以合并或拆分分段。

中间区域用于检查：

- 代表帧和缩略图
- 亮度变化曲线
- 疑似暗帧、闪烁帧和异常帧
- 手动剔除的坏帧

### 3. 选择调色方案

每段可独立选择处理风格：

- `自然`：适合白天风光，保持真实色彩。
- `通透`：提升层次、局部对比和空气感。
- `日照金山`：强化雪山受光区域的金色与冷暖对比。

处理器会先分析整段亮度，再从原始照片一次渲染出最终 JPEG，避免多个步骤反复压缩图片。

### 4. 开始处理

点击底部“开始处理”。任务栏会显示当前阶段、文件名和进度。

处理中可以取消；取消或失败不会替换上一次成功结果。修正设置后，可选择从“重新分析并渲染”或“重新渲染”开始重跑。

### 5. 导出视频

处理完成后点击“导出视频”，选择帧率、分辨率和编码格式。默认导出为 4K、30 fps、H.264 MP4。

视频保存在：

```text
output/
```

### 6. 归档

确认各段 JPEG 和 MP4 都完整后，点击“归档”。程序会校验文件数量和哈希，再创建时间目录：

```text
archive/YYYY-MM-DD_HHMMSS/
  manifest.json
  01_段名/
    *.jpg
  01_段名.mp4
```

MP4 位于时间目录顶层，JPEG 序列位于对应分段目录。归档只清理本项目的工作区和输出区，永远不会删除外部原始照片。

历史页面可以查看每次归档的分段、帧数、参数和成片。

## 项目目录

```text
config/      默认设置；本机覆盖设置不提交 Git
src/         照片分析、处理、视频导出和归档程序
webui/       浏览器界面
workspace/   当前任务、缩略图和 JPEG 处理结果
output/      当前导出的 MP4
archive/     按日期时间保存的历史成果
```

`workspace`、`output` 和 `archive` 都是本地数据目录，不会提交 Git。

## 常见问题

### 双击后提示找不到 Python

安装 Python 3.12，并确认命令行执行 `python --version` 能显示版本号，然后重新运行 `run.bat`。

### 首次安装依赖失败

检查网络后删除未完成的 `.venv` 目录，再运行 `run.bat`。也可以在项目目录执行：

```powershell
python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 浏览器没有自动打开

保持启动窗口开启，手动访问 `http://127.0.0.1:9501/`。

### 端口 9501 已被占用

关闭之前启动的 WebUI 窗口，或在“设置”中修改端口并重启。

### RAW 处理较慢

RAW 解码和 4K 导出会占用较多 CPU、内存和磁盘空间。建议先用预览和亮度曲线确认参数，再处理整段素材。
